# OpenClaw-本地部署与优化
在 Windows 11 消费级硬件上，独立完成开源 AI 智能体 OpenClaw 的本地化部署，并解决内存溢出等工程问题，实现纯本地、零成本的 AI 助手。
# 项目背景

OpenClaw 是一款开源、本地优先的 AI 智能体执行框架。与依赖云端 API 的闭源方案不同，它允许用户在个人电脑上部署强大的 AI 助手，数据完全私有。

本项目目标是：在 **16GB 内存 + RTX 4060 8G** 的笔记本上，搭建立足于本地模型的 OpenClaw 服务。

##  部署核心步骤

1.  **环境准备**：安装 Node.js (≥22) 和 Ollama，作为 OpenClaw 的运行时和模型服务。
2.  **安装 OpenClaw**：使用 `ollama launch openclaw` 一键完成从下载、安装到初始化的全过程。
3.  **模型配置**：在配置向导中选择本地 Ollama 模型，跳过所有云端 API 选项。
4.  **启动服务**：启动 OpenClaw Gateway 后，通过浏览器访问 `http://localhost:18789` 的 Web 控制台。

##  挑战与解决：内存溢出 (OOM)

在部署过程中，系统反复报错：`model requires more system memory (20.3 GiB) than is available (15.2 GiB)`。甚至更换更轻量的 `phi4-mini:3.8b` 后问题依旧，这证明是其他原因导致的内存超限。

通过分析进程和查阅资料，确认是 **OpenClaw 在调用 Ollama 时请求了过大的上下文窗口 (Context Window)**，导致系统预留内存激增。

**解决方案**：通过设置系统环境变量 `OLLAMA_CONTEXT_LENGTH`，为模型请求的上下文大小设定硬性上限。

```bash
# 在系统环境变量中新增
变量名：OLLAMA_CONTEXT_LENGTH
变量值：4096
```
